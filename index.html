Weiner Run

Weiner Run is an infinite runner mobile game created in Unity and C#. The goal of the game is to jump on the grill platforms for as long as possible and not get “sliced”. The game features a global “weinerboard”, costumes, power-ups, and an in-game currency. I created all the graphics, animations, and code for the game.

Challenges

Weiner Run was my first experience with Unity and C#. The biggest challenge I faced was making the game as efficient as I could. At first the game was terribly slow but as I refactored the code I found different programming patterns that worked better and got the game to be really efficient. I put the game on a Raspberry Pi running android and the game ran with no lag and was incredibly smooth.

Mucho Lucho

Mucho Lucho is a fighter mobile game made in Unity and C#. In the game you fight against an AI in a ring. The game features a global leaderboard holding the fighting records of all players, a basic AI, and way to customize your fighter. I created all art, animations, and code for the game.

Henry J's Taco House

Henry J’s Taco House is a restaurant located on the South Side of Des Moines, Iowa. It is owned by my uncle and has been around for about 40 years.

The Problem

Currently they package the hot sauce into styrofoam cups with plastic lids. The styrofoam and plastic are easy to break, don’t look aesthetically pleasing, and make a mess when you try to pour the hot sauce out.

My Solution

First I started with multiple rough sketches of what I wanted the basics of the design to look like. Next I moved into Illustrator and created a couple designs. Then I got two glass bottles that the hot sauce would be stored in.

Human Computer Interaction Project

This app design is for a class project I did in a human computer interaction class at Drake University. We were tasked with figuring out a solution on how online classes could be improved.

As a group we conducted interviews on parents and college-age students. From the information we gathered, we created personas, brainstormed ideas, and then created storyboards over those ideas. Next, we picked the best idea based on the storyboards and created a low-fidelity paper prototype.

Then we created user aspect reports (UARs) for the prototype and did some user testing. Using the UARs I created a high-fidelity prototype using Adobe XD. My group members then completed a heuristic evaluation and a cognitive walk through of the prototype.
